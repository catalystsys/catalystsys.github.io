Create a new Azure Active Directory app

We need an app to write/read the tables for us. Create a new one from here.

After giving it a name, we need to grant it access for the CRM, locate the CRM permission

Enable it and select enabling user_impersonation.

To be able to run the code locally (read/write) from your computer, we need to have your computer at the MSAL authentication list.

To do this, in the sidebar of the app, select Authentication > + Add a platform > Mobile and desktop applications > Add the url http://localhost

Scroll down to Advanced settings and select Yes to Allow public client flows.

Finally we need to get the app information, In the sidebar, click on Overview in the app sidebar and note the following details:

  • client id
  • client secret

In case if there is none, create a one. Also in case of that you lost your id value, delete it and create a new one (that you can store its value).

Let's write some code now, replace the values you collected in the following block

import { DynamicsWebApi } from "dynamics-web-api";
import * as dwa from 'dynamics-web-api';
import * as MSAL from '@azure/msal-node';

// put the Directory (tenant) ID after the slash here
const authorityUrl = 'https://login.microsoftonline.com/7e985654-ec67-43d9-be32-32ea313dc5e0';

const msalConfig = {
    auth: {
        authority: authorityUrl,
        // your Application (client) ID
        clientId: 'b28aff9e-bc9d-4477-bb9c-bf0b9d2f3bfc',
        // Client Secret
        clientSecret: 'Mib8Q~ZP1qoc_1HPgxdMAnaTdZ3uxkSseWktYa~f',
        knownAuthorities: ['login.microsoftonline.com']
    }
}

const cca = new MSAL.ConfidentialClientApplication(msalConfig);

// your Environment URL
const serverUrl = 'https://org83109c97.crm4.dynamics.com';

//function that acquires a token and passes it to DynamicsWebApi
const acquireToken = async () => {
    try {
        return cca.acquireTokenByClientCredential({
            scopes: [`${serverUrl}/.default`],
        });
    }
    catch (error) {
        return null;
    }
}

const dynamicsWebApi = new DynamicsWebApi({
    serverUrl: serverUrl,
    onTokenRefresh: acquireToken
});

Now finally let's model the table to insert:

async function main(): Promise<void> {
    interface newtab {
        cre3f_c?: string,
        cid?: string
    }

    //init an object representing Dynamics 365 entity
    const lead: newtab = {
        cre3f_c: "w11"
    };

    //init DynamicsWebApi request
    const request: dwa.CreateRequest = {
        collection: "cre3f_newtables",
        data: lead,
        returnRepresentation: true
    }
    const result = await dynamicsWebApi.create<newtab>(request);

}


try {
    main()
}
catch (error) {
    console.log(error);
}

Note:

cre3f_c is the c property real name, and the cid is a required value for the primary key (but we do not fill it), if i had a table named "names" i'd have it instead namesid, and so on.

For retrieving, deleting and updating the data, it is just as simple, we will just use instead the corresponding functions, for example, if we to retriev some row we would use const result = await dynamicsWebApi.retrieve<newtab>(request); instead. The package provide clear examples for doing so https://github.com/AleksandrRogov/DynamicsWebApi?tab=readme-ov-file#retrieve-a-table-row
